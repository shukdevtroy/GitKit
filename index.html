Security

Your credentials stay
where they belong.

Built on one principle: your token never leaves your machine.

Local credential storage

Your GitHub token is stored only on your device and is never transmitted to any GitKit server. GitKit talks directly to GitHub's API.

Minimal token scopes

GitKit requests only the GitHub API scopes it actually uses — repo access and user info. Nothing broader than what's needed.

No cloud backend

GitKit has no cloud intermediary. Requests go from your machine directly to GitHub's API — nothing passes through our servers.

FAQ

Common questions

What is GitKit?
GitKit is a premium desktop application that lets you manage your GitHub repositories without opening a browser. Browse repos, explore files, edit code, commit changes, and push updates — all from one focused window.
Is my GitHub token secure?
Yes. Your Personal Access Token is stored locally on your machine and is never transmitted to any GitKit server. GitKit communicates directly with GitHub's API — there is no cloud intermediary.
Does GitKit support private repositories?
Yes. If your Personal Access Token has the appropriate repo scope, GitKit can list, browse, and edit both public and private repositories you own or have been granted access to.
Can I edit files directly?
Yes. GitKit includes a built-in file editor. Open any file, make your changes, write a commit message, and push — all within the same window without switching apps.
Is GitKit free?
GitKit is available as a free download. Future plans may include additional features in a paid tier, but the core functionality for individual developers will remain free.
Which operating systems are supported?
GitKit currently supports Windows. macOS and Linux builds are in active development — check back soon for updates.

Privacy Policy

Your data stays on your machine.

Last updated: June 2026
This policy describes how GitKit, a desktop application, handles data. It's written to be clear and accurate, not as a substitute for legal advice — if you need a policy tailored to specific regulations (GDPR, CCPA, etc.), have it reviewed by a professional.

Information we don't collect

GitKit doesn't operate any servers of its own, and we don't run analytics, telemetry, or crash-reporting inside the app. We never see your code, your repositories, your commits, or your GitHub username — none of it passes through us.

What's stored on your device

When you sign in, GitKit saves a small session file locally on your computer containing:

  • Your GitHub username
  • A salted hash of the local password you set (used only to unlock the app on this device — it's never sent anywhere)
  • An obfuscated copy of your GitHub Personal Access Token, used to authenticate API requests

This file lives entirely on your machine. Signing out removes it, and you can delete it manually at any time.

Network requests

The only network requests GitKit makes are directly to GitHub's API (api.github.com), using your token, to read and update repositories on your behalf. There's no GitKit backend in between, so there's no intermediary server that could see your data in transit.

Third-party services

Because GitKit communicates directly with GitHub, any data exchanged with GitHub is also subject to GitHub's own Privacy Statement. We encourage you to review it for details on how GitHub itself handles your account data.

Children's privacy

GitKit isn't directed at children, and since we don't collect data from anyone, we don't knowingly collect data from children either.

Changes to this policy

We may update this policy as GitKit evolves. Material changes will be reflected here with a new "Last updated" date.
